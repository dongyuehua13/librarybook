## 2. 用户闭环链路

```
首页 ──→ 座位列表（筛选+占用标记） ──→ 座位详情（未来预约记录）
  │                                         │
  │                                   预约此座位（已登录）
  │                                         │
  │                                   预约表单（GET）
  │                                         │
  │                                   POST AJAX 提交
  │                                    ┌──→ 成功 → Toast + "查看我的预约"
  │                                    └──→ 失败 → Toast 显示错误
  │                                         │
  │                                   我的预约（CanCancel）
  │                                         │
  │                                   AJAX 取消 → 确认弹窗 → Toast → 刷新
  │                                         │
  └── 账号切换 / 退出（导航栏始终可用）
```

**关键交互：**
- 首页统计卡片 + 登录态导航栏（已登录/未登录/管理员）
- 座位列表：按楼层+区域下拉筛选，空闲绿色/已预约灰色徽章，空状态引导，重置筛选
- 座位详情：信息卡片 + Date≥today 预约记录表
- 预约提交：GET 表单（日期/开始时间/结束时间）→ POST AJAX（含 3 条业务规则校验）
- 我的预约：状态徽章（已预约绿/已取消灰），CanCancel 逻辑，确认弹窗
- 取消：AJAX POST → Toast → 自动刷新
- 账号切换：POST 写入 Session → 重定向首页
- 退出：清除 Session → 跳转切换页

## 4. 默认补足项 / 当前假设

| 补足项 | 补足值 | 原因 |
|--------|--------|------|
| DashboardStats 聚合类 | 独立类，含 TotalActiveSeats + TodayReservations | 类型安全，避免匿名类型 |
| Logout 路由 | `GET /Home/Logout` | docs/09 未明确定义，增加显式登出 |
| 占用判定口径 | 今天该座位有"已预约"记录 → 占用 | 按 docs/09 §2.4 实现，`DISTINCT SeatId` |
| Detail 预约记录列表 | 后端过滤 Date≥today 再传递 | docs/05 PC03：过滤未来有效记录 |
| 筛选下拉值 | 楼层 {1,2,3}，区域 {自习区/安静区/电子阅览区} | 与种子数据一致，无需动态查询 |
| 业务规则 | 最长 4h、每日 2 次上限、时段重叠检测 | 按 docs/09 §3.4 实现 |
