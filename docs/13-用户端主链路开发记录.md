## 4. 默认补足项 / 当前假设

以下项目在前序文档中未明确指定，按合理假设补足：

| 补足项 | 补足值 | 原因 |
|--------|--------|------|
| DashboardStats 聚合类 | `GetDashboardStatsAsync()` 返回 `DashboardStats` | docs/09 §2.4 中 StatsService 示例直接返回匿名聚合，此处提取为具名类便于类型安全 |
| Logout 路由 | `POST /Home/Logout` | docs/09 中未明确定义注销路由，骨架阶段已有隐式退出（switch to empty userId），此处增加显式 Logout Action |
| SeatDetailViewModel 结构 | 扁平化字段（非嵌套 Seat 实体） | docs/09 示例嵌套 `Seat Seat` 属性，此处改为直接展平为 Id/SeatNumber/Floor/Area/Description/IsActive，避免视图层依赖数据实体 |
| 筛选下拉硬编码 | 楼层 {1,2,3}，区域 {"自习区","安静区","电子阅览区"} | 与种子数据一致且未来不会变动，无需动态查询 |
| 占用判定口径 | 今天该座位有任何一条状态为"已预约"的预约记录 | docs/09 §2.4 采用 `DISTINCT SeatId` 查今天全部已预约座位，此处照此实现 |
| Detail 预约记录列表 | 显示 Date≥today 的全部记录（含已取消） | docs/05 PC03 §后端关键点："返回全部预约记录，前端根据日期筛选展示未来有效记录"。此处后端直接过滤再传递 |
