## 3. 做得好的地方

1. **分层彻底**：Controller 不直调 DbContext，全部通过 Service 接口调用——即使 Sprint 3 审计也未发现明显违规。
2. **文档持续同步**：每个 Sprint 的开发记录、任务板、README 同步更新，无断层。
3. **前序审计不回避**：发现 Sprint 2 任务板被错误标记为 100% 时立即纠正并重建。
4. **缺陷发现的时机**：Playwright 自动点击首次运行即暴露 AdminAuthFilter 无限重定向（此前代码审计未发现运行时行为）。
5. **种子数据贴近真实**：座位有明确的楼层/区域/描述，适合课堂演示。

## 5. 如果再给 1-2 次课，最该补什么

| 优先级 | 建议 | 理由 |
|--------|------|------|
| P1 | 清理重复项目目录 | 移除 `src/LibrarySeatReservation.Web/` 减少混淆 |

## 6. 最终演示顺序建议

```
用户端链路（约 5 分钟）：
  1. 打开 http://localhost:5002
  2. 首页：统计卡片 + 区域分布 + 选择体验账号
  3. 点击"张三" → 切换为张三
  4. 浏览座位 → 筛选楼层=1 + 区域=自习区 → 查看空闲/已预约标记
  5. 点击某座位"查看详情" → 信息卡片 + 未来预约记录
  6. 点击"预约此座位" → 选今天 09:00-10:00 → 提交 → 绿色 toast
  7. 再次提交同一座位同一时段 → 红色 toast"已被预约"
  8. 我的预约 → 查看刚创建的记录 → 点击"取消" → 确认 → 状态变"已取消"

管理端链路（约 4 分钟）：
  9. 访问 http://localhost:5002/Admin/Login → 输入 admin → 登录
  10. 座位管理：表格展示全部 15 个座位（含禁用座位）
  11. 点击某座位"禁用" → 行内即时切换为禁用（红色徽章）
  12. 返回首页 → 该座位不再出现在用户端列表
  13. 预约管理：选择状态=已预约 → 查看筛选结果
  14. 数据统计：查看总座位数、总预约次数、今日预约、热度排行、14 日趋势

收起（约 1 分钟）：
  15. 演示 E2E 测试：cd e2e → npx playwright test --reporter=list (12 passed)
  16. 演示烟雾测试：powershell -File smoke-test.ps1 (4/4 passed)
  17. 演示仓库状态：git log --oneline → 展示完整提交历史
```
