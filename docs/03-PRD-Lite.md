## 1. 项目概述

做一个轻量化的图书馆座位预约网页系统，解决学生找座难、占座乱、管理员管座累的问题。学生可以查看空闲座位、预约时段、取消预约；管理员可以维护座位信息、查看预约记录和统计数据。

这是单人课堂实训项目，技术栈固定为 ASP.NET Core MVC (.NET 8) + Razor + SQL Server LocalDB + EF Core + Bootstrap，不做第三方依赖。

## 2. 项目目标

1. 学生能在线看到空闲座位，选时段预约，不想去了能自己取消；
2. 管理员能登录后台新增/编辑/禁用座位，查看所有预约记录和使用数据；
3. 页面用 Bootstrap 响应式布局，电脑和手机都能正常显示；
4. 数据存在本地 LocalDB，`dotnet run` 一键启动，不需要额外配置。

## 3. 目标用户

| 用户角色 | 身份 | 权限范围 |
|---------|------|---------|
| 在校学生 | 使用自习座位的本科生/研究生 | 查看座位、预约、取消自己的预约 |
| 图书馆管理员 | 负责座位和预约管理的工作人员 | 管理座位、查看所有预约、看统计数据 |

仅此两类用户，没有第三类角色。

## 4. 使用场景

**场景一：学生预约座位**
大三学生张明早上到图书馆想找座位自习。他打开系统，在座位列表看到 3 楼 C-03 显示"空闲"，点进详情确认座位有电源插座，选择"今天 9:00-12:00"提交预约。系统提示预约成功，他到馆直接入座。

**场景二：学生取消预约**
张明临时被导师叫去开会，去不了图书馆。他打开"我的预约"，找到今天的预约记录，点击取消。预约状态变为"已取消"，座位释放给其他人预约。

**场景三：管理员维护座位**
管理员李老师发现 1 楼 A-02 座位台灯坏了。他登录后台，在座位管理页找到 A-02，将其设为"禁用"。修好后再改回"启用"。

**场景四：管理员看数据**
馆长想了解本周座位使用情况。管理员登录后台进统计页，看到座位热度排行和近 14 日预约趋势，发现 2 楼安静区预约率最高。

## 5. 用户端核心功能

### F1 首页

|------|------|
| 功能描述 | 系统入口页，展示概况数据和导航入口 |
| 页面要素 | 系统标题、总座位数卡片、今日预约数卡片、楼层分布卡片、"浏览座位"和"我的预约"入口按钮、当前用户提示 |
| 前置条件 | 无（无需登录即可访问） |
| **最小验收标准** | ① 首页能正常打开，显示三个统计数据卡片 ② 未选择账号时"我的预约"按钮置灰 ③ 选择账号后显示用户名和退出按钮 |

### F2 座位列表

| 项目 | 内容 |
|------|------|
| 功能描述 | 展示全部可用座位，区分空闲/已预约状态，支持筛选 |
| 页面要素 | 楼层下拉筛选、区域下拉筛选、重置按钮、座位卡片网格（编号、楼层、区域、状态徽章、描述） |
| 前置条件 | 无 |
| 业务规则 | 今日有有效预约的座位标记为"已预约"（红色徽章），否则为"空闲"（绿色徽章）；仅显示已启用的座位 |
| **最小验收标准** | ① 页面展示至少 10 个座位卡片 ② 空闲座位显示绿色"空闲"徽章 ③ 按楼层筛选后只显示该楼层座位 ④ 按区域筛选后只显示该区域座位 ⑤ 重置按钮清除筛选条件 |

### F3 座位详情

| 项目 | 内容 |
|------|------|
| 功能描述 | 查看单个座位的完整信息和预约记录 |
| 页面要素 | 座位信息表（编号、楼层、区域、描述、状态）、预约记录列表（日期、时段、状态）、"预约此座位"按钮 |
| 前置条件 | 座位 ID 有效且已启用 |
| **最小验收标准** | ① 显示座位完整信息 ② 显示该座位的预约记录列表 ③ "预约此座位"按钮点击后进入预约提交页 ④ 未选择账号时预约按钮置灰并提示选账号 |

### F4 预约提交

| 项目 | 内容 |
|------|------|
| 功能描述 | 选择日期和时段，提交预约申请 |
| 页面要素 | 座位基本信息（只读）、日期选择器、开始时间下拉（8:00-21:00）、结束时间下拉（9:00-22:00）、提交按钮 |
| 前置条件 | 已选择学生账号，座位有效 |
| 业务规则 | ① 时段冲突检测：同一天同座位已有时段重叠的有效预约则拒绝；开始时间必须早于结束时间 ② 单次预约最长 4 小时 ③ 同一学生每天最多 2 次有效预约 ④ 同一学生不可在同一时段预约多个座位 |
| **最小验收标准** | ① 选择日期和时段后提交，提示"预约成功" ② 同一座位同一天同一时段重复预约，提示"该时段已被预约" ③ 预约时长超过 4 小时提示"单次预约最长4小时" ④ 当天已预约 2 次提示"当日预约已达上限" ⑤ 预约成功后按钮置灰不可重复提交 ⑥ 预约数据存入数据库，可在"我的预约"中查到 |

### F5 我的预约（含取消）

| 项目 | 内容 |
|------|------|
| 功能描述 | 展示当前学生的全部预约记录，并提供取消入口 |
| 页面要素 | 预约记录表格（座位编号、日期、时段、状态、预约时间）、取消按钮 |
| 前置条件 | 已选择学生账号 |
| 展示规则 | 按创建时间倒序排列 |
| 取消规则 | 仅状态为"已预约"的可取消；日期在今天之前的不可取消；日期为今天且开始时间已过的不可取消 |
| **最小验收标准** | ① 显示当前学生的所有预约记录 ② 已预约的记录显示绿色"已预约"徽章 ③ 可取消的记录显示红色"取消"按钮 ④ 点击取消后弹出确认框，确认后状态变为"已取消" ⑤ 已过期的预约不显示取消按钮 |

### F6 模拟账号切换

| 项目 | 内容 |

| 功能描述 | 无需注册登录，直接选择学生账号模拟登录 |

## 8. 核心业务流程

```
流程一：学生预约座位
首页 → (选择账号 → 提示嵌入在导航栏) → 座位列表 → (按楼层筛选) → 座位详情 → 预约提交 → (选日期时段) → 提交成功 → 我的预约 → 确认记录存在

流程二：学生取消预约
我的预约 → 找到目标记录 → 点击取消 → 确认弹窗 → 状态变为"已取消"

流程三：管理员管理
管理员登录 → 座位管理 → (新增/编辑/禁用) → 预约管理 → (按状态筛选) → 数据统计 → (查看热度排行和趋势)
```

## 12. 状态流转说明

### 12.1 预约状态流转

```
[学生提交预约]          [学生点击取消]
      │                     │
      ▼                     ▼
  已预约 ──────────────→ 已取消
  (初始状态)           (终态，不可逆)
```

- 只有状态为"已预约"的记录可以取消
- 已取消的记录不能恢复
- 没有其他中间状态（不做"待审核""进行中""已完成"等）

### 12.2 座位状态流转

```
[管理员点击启用/禁用]
         │
         ▼
  启用 ←────────────→ 禁用
(IsActive = true)    (IsActive = false)
```

- 已禁用的座位不在前台座位列表中显示
- 已存在的预约记录不受座位禁用影响

## 15. 风险与假设

### 风险

| 风险 | 可能性 | 应对 |
|------|--------|------|
| 时间不够，开发进度落后 | 高 | 严格按开发顺序执行，P0 优先，P1 视时间决定 |
| EF Core 关联查询报错卡住 | 中 | 三张表关系简单，不做复杂查询，必要时用原生 SQL |
| AI 生成的代码存在兼容性问题 | 中 | 每完成一个功能立刻手动测试，不堆到后面一起排错 |
| Bootstrap 样式调不好影响演示效果 | 中 | 直接使用 Bootstrap 官方组件和配色，不自定义复杂样式 |
| LocalDB 连接失败或数据库文件损坏 | 低 | 使用 `EnsureCreated` 自动建库，无需手动管理数据库文件 |

### 假设

1. **单楼层单区域数据模型**：不做多馆多区联动，楼层和区域仅作为筛选标签；
2. **预约即生效**：提交预约直接成功，无需管理员审核，不设黑名单；
3. **无惩罚机制**：学生取消预约无次数限制，不做违规惩罚逻辑；
4. **本地环境运行**：全程使用 LocalDB，不部署远程服务器，不配置 IIS；
5. **单人开发**：无团队协作，无代码审查，无 CI/CD，范围自己控制。
