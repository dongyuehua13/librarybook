---

## 2. 实现决策

| 决策项 | 选择 | 理由 |
|--------|------|------|
| 权限实现 | `IActionFilter`（代码层面） | 简化实现，在 OnActionExecuting 中通过 RouteData 判断 Action 名称来豁免 Login/Logout。功能等价于 `IAuthorizationFilter`，代码更紧凑 |
| 管理端 Session Key | `AdminId` / `AdminName` | 与用户端 `UserId` / `UserName` 隔离，互不干扰 |
| 座位启用/禁用交互 | AJAX POST + 行内 DOM 更新 | 通过 `ToggleActiveAsync` 切换 `Seat.IsActive`，前端不整页刷新 |
| 统计口径 | 全部基于 EF Core 实时查询 | 不使用缓存或硬编码，确保数据即时反映 DB 状态 |
| 管理端布局 | `_AdminLayout.cshtml` | 独立于用户端 `_Layout.cshtml`，导航栏仅包含管理端页面入口 |

## 3. 实现结构

### 3.1 AdminController（`Controllers/AdminController.cs`）

| Action | 路由 | 方法 | 主要功能 |
|--------|------|------|---------|
| Login | GET/POST `/Admin/Login` | `Login()` | 显示登录表单 / 验证用户名 + 写入 Session |
| Logout | POST `/Admin/Logout` | `Logout()` | 清除 Session → 重定向 Login |
| Seats | GET `/Admin/Seats` | `Seats()` | 展示全部座位表格 |
| SeatCreate | GET/POST `/Admin/SeatCreate` | `SeatCreate()` | 新增座位表单（含编号查重） |
| SeatEdit | GET/POST `/Admin/SeatEdit/{id}` | `SeatEdit()` | 编辑座位表单（含排除自身查重） |
| SeatToggle | POST `/Admin/SeatToggle` | `SeatToggle()` | AJAX 启用/禁用座位 |
| Reservations | GET `/Admin/Reservations` | `Reservations()` | 预约记录筛选与展示 |
| Stats | GET `/Admin/Stats` | `Stats()` | 统计面板展示 |

### 3.2 AdminAuthFilter（`Filters/AdminAuthFilter.cs`）

- 注册为 Scoped 服务（`services.AddScoped<AdminAuthFilter>()`）
- 类级别 `[ServiceFilter(typeof(AdminAuthFilter))]` 标记在 Controller 上
- 拦截逻辑：
  - 若 Action 为 `Login` / `Logout` → 跳过（return）
  - 若 `Session["AdminId"]` 不存在 → 普通请求重定向 `/Admin/Login`；AJAX 请求返回 `401 Unauthorized` JSON
  - 若已登录访问 Login → 自动跳转 `/Admin/Seats`

### 3.3 视图文件（`Views/Admin/`）

| 文件 | 说明 |
|------|------|
| Login.cshtml | 用户名输入框 + 错误信息显示 |
| Seats.cshtml | 座位表格 + 行内 AJAX 切换 |
| SeatCreate.cshtml | 新增座位表单 |
| SeatEdit.cshtml | 编辑座位表单（回填现数据） |
| Reservations.cshtml | 预约记录表 + 筛选下拉 + 日期范围 |
| Stats.cshtml | 统计卡片 + 座位热度排行 + 预约趋势 |

### 3.4 后端支撑

- `SeatService.ToggleActiveAsync(int id)` → 切换 `IsActive`，返回 `(Success, Message, IsActive)`
- `SeatService.CreateSeatAsync(Seat)` → 含编号查重
- `SeatService.UpdateSeatAsync(Seat)` → 含排除自身查重
- `ReservationService.GetAllReservationsAsync(status, dateFrom, dateTo)` → 三条件组合筛选
- `StatsService.GetAllStatsAsync()` → 四项实时聚合查询

## 4. 管理端 → 用户端状态回流机制

| 管理端操作 | 用户端效果 | 实现位置 |
|-----------|-----------|---------|
| 禁用座位 | 用户端座位列表不再显示该座位 | `SeatService.GetSeatsWithStatusAsync` 过滤 `s.IsActive == true` |
| 重新启用座位 | 用户端恢复显示 | 同上 |
| 新增座位 | 用户端可见（如为启用状态） | 直接写入同一 DB |
| 编辑座位信息 | 用户端详情页同步更新 | 同一 DB 数据源 |
| 查看预约 | 展示用户端创建的全部预约记录 | `ReservationService.GetAllReservationsAsync` |
| 统计页 | 聚合用户端全部操作数据 | `StatsService.GetAllStatsAsync` |
