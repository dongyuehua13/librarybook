## 1. 系统设计目标

1. **轻量可演示** — 单人课堂项目，不做分布式、不做微服务、不做消息队列。一个 ASP.NET Core MVC 项目跑到底。
2. **分层清晰** — Controller 只做参数接收和页面跳转，业务逻辑放在 Service，数据访问走 EF Core DbContext，不混在一起写。
3. **一键运行** — `dotnet run` 启动，LocalDB 自动创建，种子数据自动写入，零配置。
4. **手机能看** — 前台 Bootstrap 响应式，后台桌面端表格优先，各自适配。

## 4. 分层说明

### 4.1 分层总览

```
┌─────────────────────────────────────────────────┐
│  Views (Razor .cshtml)                           │
│  只做 HTML 渲染，不写 C# 业务逻辑                 │
│  通过 @Model 读取 ViewModel 数据                  │
├─────────────────────────────────────────────────┤
│  Controllers                                     │
│  接收 HTTP 请求 → 调用 Service → 构造 ViewModel  │
│  → 返回 View() 或 Json()                         │
│  不做数据库直接查询，不做业务规则判断               │
├─────────────────────────────────────────────────┤
│  Services                                        │
│  写业务规则：冲突检测、取消条件判断、统计聚合计算    │
│  调用 DbContext 做数据读写                        │
│  可被多个 Controller 复用                         │
├─────────────────────────────────────────────────┤
│  Data (EF Core DbContext)                        │
│  实体配置、数据库连接、种子数据                     │
│  不写业务逻辑，只做 CRUD                          │
├─────────────────────────────────────────────────┤
│  Models (Entity) + ViewModels                    │
│  Entity = 数据库表映射，ViewModel = 页面数据包装    │
│  两者分开，不把 ViewModel 塞进 Models 目录         │
└─────────────────────────────────────────────────┘
```

### 4.2 各层详细职责

| 层 | 放在哪 | 职责 | 禁入规则 |
|----|--------|------|---------|
| **Controller** | `Controllers/` | ① 从路由参数/Form 读取输入 ② 调用 Service 方法 ③ 构造 ViewModel 或直接传 Model ④ `return View()` 或 `return Json()` ⑤ Session 的读写（登录校验） | ❌ 不写 SQL/EF 查询 ❌ 不写 if-else 业务规则 |
| **Service** | `Services/` | ① 预约时段冲突检测 ② 取消条件判断 ③ 统计聚合（COUNT/GROUP BY） ④ 增删改查的编排 ⑤ 返回结果用 bool + string 或 DTO | ❌ 不引用 `HttpContext` ❌ 不返回 `IActionResult` ❌ 不直接操作 Session |
| **Data** | `Data/` | ① DbContext 的 `DbSet<>` 属性 ② `OnModelCreating` 的实体关系配置 ③ `DbInitializer.Seed()` 种子数据 | ❌ 不写业务判断 ❌ 不引用 Controller/Service |
| **View** | `Views/` | ① 用 Razor 语法渲染 HTML ② 用 `@Model` 读取数据 ③ 用 `ViewBag` 传递零星数据 ④ 包含 JS 片段做交互 | ❌ 不写数据库查询 ❌ 不写复杂 C# 逻辑（超过 3 行就放 Service） |
| **ViewModel** | `ViewModels/` | ① 聚合多表数据给一个页面用 ② 包含 SelectListItem 供下拉框用 ③ 包含展示用的布尔标记（如 `CanCancel`） | ❌ 不加 `[Key]` 等数据库特性 ❌ 不作为 EF Core 查询对象 |

### 4.3 为什么需要 ViewModel 层

直接原因：一个页面通常需要多个 Entity 的数据。

**示例：座位详情页需要的数据**
```
Seat 表 → 座位编号、楼层、区域、描述
Reservations 表 → 该座位的预约记录列表（多条）
→ 包装成 SeatDetailViewModel { Seat, List<Reservation> }
```

不绕开 ViewModel 的规则：
- 如果页面只展示一个 Entity 的数据（如编辑座位表单），可以直接用 Entity 作为 `@Model`
- 如果页面展示多个来源的数据（如首页 + 统计页），必须建 ViewModel

### 4.4 ViewModel 示例

```csharp
// ViewModels/SeatDetailViewModel.cs
public class SeatDetailViewModel
{
    public Seat Seat { get; set; }
    public List<Reservation> ReservationRecords { get; set; }
    public bool IsCurrentUserLoggedIn { get; set; }
}

// ViewModels/MyReservationsViewModel.cs
public class MyReservationsViewModel
{
    public List<ReservationItem> Items { get; set; }
    public bool HasReservations => Items.Any();
}

public class ReservationItem
{
    public int Id { get; set; }
    public string SeatNumber { get; set; }
    public DateTime Date { get; set; }
    public string TimeSlot { get; set; }
    public string Status { get; set; }
    public DateTime CreatedAt { get; set; }
    public bool CanCancel { get; set; }  // Service 层计算好，View 直接用
}
```

## 7. 主链路工程落点

### 7.1 学生预约链路

```
浏览器 GET / → HomeController.Index()
  → StatsService.GetDashboardStats()
    → DbContext.Seats.CountAsync()    ← 总座位数
    → DbContext.Reservations.CountAsync(r => r.Date == today && r.Status == "已预约") ← 今日预约数（仅有效预约）
  → return View(viewModel)

浏览器 GET /Home/Seats → HomeController.Seats(floor?, area?)
  → SeatService.GetSeatsWithStatus(floor, area)
    → 查询 Seats (IsActive=true)
    → 查询当天有效预约的 SeatId 集合
    → 拼接每个座位的预约状态标记
  → return View(seats)

浏览器 GET /Home/Detail/1 → HomeController.Detail(1)
  → SeatService.GetSeatDetail(1)
    → _db.Seats.Find(1)
    → _db.Reservations.Where(r => r.SeatId == 1).OrderBy().ToList()
  → return View(viewModel)

浏览器 GET /Home/Reserve/1 → HomeController.Reserve(1)
  → 校验 Session 中 UserId 是否存在
  → SeatService.GetSeatById(1) → return View(seat)

浏览器 POST /Home/Reserve → HomeController.Reserve(seatId, date, start, end)
  → ReservationService.Create(userId, seatId, date, start, end)
    → 冲突检测：检查同一日期同一座位时段重叠的有效预约
    → if (冲突) return { success: false, msg: "该时段已被预约" }
    → else _db.Reservations.Add() + SaveChanges() → return { success: true }
  → Controller 返回 JsonResult

浏览器 GET /Home/MyReservations → HomeController.MyReservations()
  → 从 Session 取 UserId
  → ReservationService.GetUserReservations(userId)
    → _db.Reservations.Where(r => r.UserId == userId).Include(r => r.Seat).ToList()
    → 遍历每条记录，计算 CanCancel 标记
  → return View(viewModel)
```

### 7.2 管理链路

```
浏览器 POST /Admin/Login → AdminController.Login(username)
  → UserService.ValidateAdmin(username)
    → _db.Users.FirstOrDefault(u => u.Username == username && u.IsAdmin)
  → if (user == null) return View with error
  → else Session.Set("AdminId") → Redirect to /Admin/Seats

浏览器 GET /Admin/Seats → AdminController.Seats()
  → 校验 Session 中 AdminId
  → SeatService.GetAllSeats()
    → _db.Seats.OrderBy(s => s.SeatNumber).ToList()
  → return View(seats)

浏览器 POST /Admin/SeatToggle → AdminController.SeatToggle(id)
  → SeatService.ToggleActive(id)
    → _db.Seats.Find(id) → IsActive = !IsActive → SaveChanges()
  → return Json({ success: true })

浏览器 GET /Admin/Stats → AdminController.Stats()
  → StatsService.GetAllStats()
    → 总座位数、总预约数、今日预约数
    → 各座位预约计数（GROUP BY, ORDER BY DESC）
    → 近14日每日预约计数（GROUP BY Date）
  → 全部存入 ViewBag → return View()
```

## 8. 异常处理原则

### 8.1 处理策略

| 异常类型 | 处理方式 | 示例 |
|---------|--------|------|
| 数据不存在 | 返回 404 友好页面，提供回退链接 | 无效座位 ID → "座位不存在" + 返回列表链接 |
| 参数不合法 | 前端 JS 校验拦截 + 后端二次校验 | 结束时间 ≤ 开始时间 → 提示"请选择有效时段" |
| 业务规则冲突 | Service 返回 bool + 错误信息，Controller 返回 Json | 预约时段冲突 → `{success:false, msg:"该时段已被预约"}` |
| Session 过期 | 跳转到账号选择页或管理登录页 | 预约提交时 Session 无 UserId → 跳转 SwitchUser |
| 未授权访问 | 跳转到登录页 | 未登录访问 `/Admin/Seats` → 重定向到 `/Admin/Login` |
| 数据库连接失败 | 页面显示"数据加载失败"，不暴露堆栈 | 首页统计卡片显示"—" |

### 8.2 不做的异常处理
- 不做全局异常过滤器（页面少，手动处理即可）
- 不做日志记录到文件（控制台输出够用）
- 不做 500 错误页（开发环境直接看堆栈）

### 8.3 Session 校验写法

```csharp
// 每个需要登录的 Action 第一行加校验
// AdminController 示例：
public IActionResult Seats()
{
    if (HttpContext.Session.GetInt32("AdminId") == null)
        return RedirectToAction("Login");
    // ... 正常逻辑
}

// 或者抽取成 Filer，统一应用到 AdminController
// Filters/AdminAuthFilter.cs
public class AdminAuthFilter : IActionFilter
{
    public void OnActionExecuting(ActionExecutingContext context)
    {
        if (context.HttpContext.Session.GetInt32("AdminId") == null)
            context.Result = new RedirectToActionResult("Login", "Admin", null);
    }
    public void OnActionExecuted(ActionExecutedContext context) { }
}
```

## 9. Session / Cookie 使用原则

### 9.1 Session 存储内容

| Key | 类型 | 写入时机 | 清除时机 | 用途 |
|-----|------|---------|---------|------|
| `UserId` | int | 学生切换账号时 | 退出时 | 标识当前登录学生 |
| `UserName` | string | 学生切换账号时 | 退出时 | 页面顶部显示用户名 |
| `AdminId` | int | 管理员登录成功时 | 退出时 | 标识当前登录管理员 |
| `AdminName` | string | 管理员登录成功时 | 退出时 | 后台显示管理员名 |

### 9.2 原则

| 原则 | 说明 |
|------|------|
| 不存密码 | 系统无密码，管理员仅校验 Username |
| Session 超时 | 默认 20 分钟，演示时可在 `Program.cs` 中改为 4 小时 |
| 不依赖 Cookie 做权限 | Session 服务端存储，客户端只有加密 Session ID |
| 不做 Remember Me | 课堂项目无需持久化登录态 |
| 前后台 Session 隔离 | UserId 是学生身份，AdminId 是管理员身份，两者互不覆盖 |

### 9.3 Program.cs 配置

```csharp
builder.Services.AddDistributedMemoryCache();
builder.Services.AddSession(options =>
{
    options.IdleTimeout = TimeSpan.FromHours(4);
    options.Cookie.HttpOnly = true;
    options.Cookie.IsEssential = true;
});

// ... 中间件管道中
app.UseSession();
```
