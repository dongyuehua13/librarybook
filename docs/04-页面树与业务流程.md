## 4. 用户主链路

### 4.1 链路一：学生预约座位（闭环路径）

```
用户首页 ──→ 座位列表页 ──→ 座位详情页 ──→ 预约提交页 ──→ 我的预约页
  ▲                                                         │
  └─────────────────────────────────────────────────────────┘
                          (返回首页)
```

**步骤拆解：**

| 步骤 | 页面 | 用户动作 | 系统响应 |
|------|------|---------|---------|
| 1 | 用户首页 | 查看概况卡片，点击"浏览座位" | 跳转到座位列表页 |
| 2 | 座位列表页 | 按楼层/区域筛选，点击空闲座位的"查看详情" | 跳转到座位详情页 |
| 3 | 座位详情页 | 查看座位信息，点击"预约此座位" | 如未选账号则提示切换，已选则跳转预约提交页 |
| 4 | 预约提交页 | 选择日期、开始时间、结束时间，点击"提交预约" | 冲突检测 → 预约成功 JSON 响应 |
| 5 | 我的预约页 | 点击结果提示中的"查看我的预约"或导航栏进入 | 显示刚创建的预约记录，状态为"已预约" |
| 6 | 用户首页 | 点击导航栏"首页" | 返回首页，首页今日预约数更新 |

### 4.2 链路二：学生取消预约（闭环路径）

```
                 我的预约页
                     │
                     ▼
               (找到目标记录)
                     │
                     ▼
               (确认可取消条件)
                     │
                     ▼
               (点击取消按钮)
                     │
                     ▼
               (确认弹窗)
                     │
                     ▼
               (状态更新为已取消)
                     │
                     ▼
               (页面刷新，显示新状态)
                     │
                     ▼
               (返回首页或留在当前页)
```

**步骤拆解：**

| 步骤 | 页面 | 用户动作 | 系统响应 |
|------|------|---------|---------|
| 1 | 我的预约页 | 浏览预约记录列表，找到目标记录 | 显示每条记录的状态和取消按钮状态 |
| 2 | 我的预约页 | 点击记录行上的"取消"按钮 | 弹出确认弹窗："确定取消该预约？" |
| 3 | 我的预约页 | 点击确认弹窗中的"确定" | POST 请求，服务端校验 → 状态更新为"已取消" |
| 4 | 我的预约页 | 页面自动刷新 | 该记录徽章变为灰色"已取消"，取消按钮消失 |

### 4.3 链路完整性验证

| 检查项 | 是否闭环 | 说明 |
|--------|---------|------|
| 起点 → 终点 | ✅ | 首页开始，完成预约，回到首页 |
| 数据一致性 | ✅ | 预约数据写入 DB，我的预约页读取同一 DB，数据一致 |
| 异常回退 | ✅ | 冲突预约有明确提示，可返回修改时段 |
| 空状态处理 | ✅ | 无预约时显示引导文字"去预约座位" |

## 6. 状态流转

### 6.1 预约状态流转图

```
                        ┌─────────────┐
                        │             │
         ┌──────────────┤  开始预约    │
         │              │             │
         │              └──────┬──────┘
         │                     │
         │                     ▼
         │              ┌─────────────┐
         │              │             │
         │              │  已预约      │  ← 初始状态，预约成功后进入
         │              │             │
         │              └──────┬──────┘
         │                     │
         │                     │  学生点击取消（需满足时间条件）
         │                     ▼
         │              ┌─────────────┐
         │              │             │
         └──────────────┤  已取消      │  ← 终态，不可逆
                        │             │
                        └─────────────┘
```

**状态说明：**

| 状态 | 含义 | 进入条件 | 可操作 |
|------|------|---------|-------|
| 已预约 | 预约有效，座位已被占用 | 学生提交预约，无时段冲突 | 可取消 |
| 已取消 | 预约作废，座位释放 | 学生在可取消时间内点击取消 | 无（终态） |

**不可取消的条件（取消按钮隐藏或置灰）：**
```
预约状态 != "已预约"        → 不可取消
预约日期 < 今天              → 不可取消（已过期）
预约日期 == 今天 AND 开始时间 <= 当前时间 → 不可取消（已开始）
```

### 6.2 座位状态流转图

```
                        ┌─────────────┐
                        │             │
                        │   启用       │  ← IsActive = true，前台可见
                        │             │
                        └──────┬──────┘
                               │
                    ┌──────────┴──────────┐
                    │                     │
                    │ 管理员点击"禁用"     │ 管理员点击"启用"
                    │                     │
                    ▼                     ▼
               ┌─────────────┐    ┌─────────────┐
               │             │    │             │
               │   禁用       │    │   启用       │
               │             │    │             │
               └─────────────┘    └─────────────┘
```

**状态说明：**

| 状态 | 含义 | 前台表现 | 预约影响 |
|------|------|---------|---------|
| 启用 | 座位可用，学生可预约 | 在座位列表中显示 | 可提交新预约 |
| 禁用 | 座位不可用，暂不开放 | 不在座位列表中显示 | 已有预约不受影响 |
