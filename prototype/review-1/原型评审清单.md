## 五、哪些问题必须在接后端前修改

| 优先级 | 问题描述 | 影响 | 修改建议 |
|--------|---------|------|---------|
| **高** | 统计页缺少无数据空状态 | 演示时若数据库无数据，统计页留白且无引导 | 在热度排行和趋势区域各补一条空状态 |
| **高** | 表单缺少自定义 invalid-feedback 样式 | 用户提交空表单时只显示浏览器默认提示，体验差 | 补充 Bootstrap invalid-feedback 结构和样式 |
| **中** | detail.html 中未登录状态下预约按钮的处理 | 当前原型假设已登录，未展示未登录置灰状态 | 增加一条示例展示未登录时按钮置灰 + 提示"请先选择账号" |
| **中** | 我的预约页桌面表格行间距略紧凑 | 长期使用可读性受影响 | 增大 td padding 至 12px 16px |
| **低** | 前端页面无 loading 状态 | 后端真实数据加载时有短暂空白 | 可在 Razor 视图开发阶段补充 |

### 优先修复清单（建议 static-v2 前完成）

```
□ 1. 统计页补充空状态示例
□ 2. 补充表单 invalid-feedback 样式和结构
□ 3. detail.html 补充未登录状态示例
□ 4. 整理原型文件命名与 MVC 路由对照表
□ 5. 测试 375px 极端窄屏下的布局溢出
```
